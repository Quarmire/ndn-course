## Five phases, from "run it" to "reason about it"

The course is a staircase against a real codebase. Each phase ends with an identity —
something you can now *say you are*.

<CardGrid>
	<Card title="Phase 0 — Orientation" icon="rocket">
		Run the forwarder; read a multi-crate workspace. **"I can run it and find things."**
	</Card>
	<Card title="Phase 1 — Bytes, memory, bugs" icon="seti:rust">
		Build a codec from scratch, borrow instead of copy, hunt planted bugs.
		**"I can build, test, and debug."**
	</Card>
	<Card title="Phase 2 — Types that carry meaning" icon="puzzle">
		Design an API, parse without copying, choose static vs dynamic dispatch.
		**"I can design and prove."**

	<Card title="Phase 3 — Threads, processes, async" icon="random">

	<Card title="Phase 4 — Architecture & NDF" icon="approve-check">

	<Card title="Two capstones" icon="star">

## How it feels

You never read the Rust Book cover to cover. You run `ndn-fwd`, poke it, and by
module 3 you've written the parser for the packets it speaks. Every exercise is
graded three ways — mechanical tests, a rubric review by an LLM tutor, and a spoken
reflection — and the whole thing is pinned to tagged snapshots so it never rots out
from under you.
